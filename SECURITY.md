# Security Policy

## Supported Versions

The following versions of the Attendance System are currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of the Attendance System seriously. If you believe you have found a security vulnerability, please report it to us responsibly.

### How to Report

Please **do not** report security vulnerabilities via public GitHub issues. Instead, send an email to **security@example.com** with the following details:

- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue (and any proof-of-concept code, if available).
- The version(s) of the software affected.

### What to Expect

- We will acknowledge receipt of your report within **48 hours**.
- We will investigate and attempt to reproduce the issue.
- If the vulnerability is confirmed, we will work on a fix and release it in a timely manner.
- We will keep you updated on our progress throughout the process.

### Disclosure Policy

- We ask that you give us a reasonable amount of time to fix the issue before disclosure.
- Once a fix is released, we will publicly disclose the vulnerability in a coordinated fashion.

Thank you for helping us keep the system secure!
